# <span data-ttu-id="d367d-103">Neuigkeiten in MED-V 2.0</span><span class="sxs-lookup"><span data-stu-id="d367d-103">What's New in MED-V 2.0</span></span>


<span data-ttu-id="d367d-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2,0 hat die Unterstützung für die Anwendungskompatibilität für Windows 7 entwickelt und Funktionen entfernt, die für dieses Szenario nicht erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="d367d-104">Microsoft Enterprise Desktop Virtualization (MED-V) 2.0 has evolved the application compatibility support for Windows 7 and removed functionality that is not required for this scenario.</span></span> <span data-ttu-id="d367d-105">Beispielsweise wurden Features wie die Verschlüsselung des Med-v-Arbeitsbereichs, der zentralisierte Med-v-Server und die Übertragung des Med-v-Arbeitsbereichs entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-105">For example, features such as encryption of the MED-V workspace, the centralized MED-V server, and MED-V workspace trim transfer have been removed.</span></span>

## <span data-ttu-id="d367d-106">Änderungen der Standard Funktionalität</span><span class="sxs-lookup"><span data-stu-id="d367d-106">Changes in Standard Functionality</span></span>


<span data-ttu-id="d367d-107">In diesem Abschnitt werden die wichtigsten Bereiche erläutert, in denen sich die MED-V 2,0-Funktionalität geändert hat.</span><span class="sxs-lookup"><span data-stu-id="d367d-107">This section discusses the key areas where MED-V 2.0 functionality has changed.</span></span>

### <span data-ttu-id="d367d-108">Erstellen eines MED-V-Arbeitsbereichs</span><span class="sxs-lookup"><span data-stu-id="d367d-108">MED-V Workspace Creation</span></span>

<span data-ttu-id="d367d-109">Die für den MED-V-Arbeitsbereich verwendete virtuelle Festplatte wird jetzt in Windows Virtual PC erstellt.</span><span class="sxs-lookup"><span data-stu-id="d367d-109">The virtual hard disk used for the MED-V workspace is now created in Windows Virtual PC.</span></span> <span data-ttu-id="d367d-110">Die zum Erstellen des MED-V-Arbeitsbereichs verwendeten Methoden umfassen die Installation von Windows XP SP3, das Aktualisieren des Betriebssystems und das Vorbereiten der Verwaltung über die Softwareverwaltungsinfrastruktur.</span><span class="sxs-lookup"><span data-stu-id="d367d-110">The methods that are used to create the MED-V workspace include installing Windows XP SP3, updating the operating system, and preparing it to be managed through software management infrastructure.</span></span>

<span data-ttu-id="d367d-111">Die Funktion für die Offlineverwaltung und die Trimm Übertragung wurde zusätzlich zur proprietären MED-V Workspace-Verschlüsselungs-und-Komprimierungsfunktion entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-111">The offline management and trim transfer functionality were removed, in addition to the proprietary MED-V workspace encryption and compression functionality.</span></span> <span data-ttu-id="d367d-112">Wenn Sie einen Med-v-Arbeitsbereich erstellen, sollte ein Administrator geeignete Anwendungen und Verwaltungstools im Abbild vorbereiten und konfigurieren, anstatt das Tool zur Vorbereitung der virtuellen Maschine zu verwenden, das in Med-v 1,0 bereitgestellt wird.</span><span class="sxs-lookup"><span data-stu-id="d367d-112">When you create a MED-V workspace, an administrator should prepare and configure appropriate applications and management tools in the image instead of using the virtual machine preparation tool that is provided in MED-V 1.0.</span></span>

<span data-ttu-id="d367d-113">Das Ausführen von Sysprep für das Med-v-Abbild ist jetzt erforderlich und wird während der Verpackung des Med-v-Arbeitsbereichs überprüft.</span><span class="sxs-lookup"><span data-stu-id="d367d-113">Running Sysprep on the MED-V image is now required and validated during the packaging of the MED-V workspace.</span></span> <span data-ttu-id="d367d-114">Der MED-V Workspace Packager bietet eine grafische Benutzeroberfläche (Graphical User Interface, GUI), die den Administrator durch den Verpackungsprozess führt.</span><span class="sxs-lookup"><span data-stu-id="d367d-114">The MED-V Workspace Packager provides a graphical user interface (GUI) that guides the administrator through the packaging process.</span></span> <span data-ttu-id="d367d-115">Die Konsole von Med-v 1,0 wurde zusammen mit der Funktionalität zum Verwalten von Bildern, Verwalten von Med-v-Arbeitsbereichs Profilen und der Anforderung zum Bereitstellen und Verschlüsseln von Med-v-Arbeitsbereichen entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-115">The console from MED-V 1.0 was removed together with the functionality of managing images, managing MED-V workspace profiles, and the requirement to stage and encrypt MED-V workspaces.</span></span>

### <span data-ttu-id="d367d-116">Bereitstellung von MED-V Workspace</span><span class="sxs-lookup"><span data-stu-id="d367d-116">MED-V Workspace Deployment</span></span>

<span data-ttu-id="d367d-117">Zur Bereitstellungeines MED-V-Arbeitsbereichs kann ein Administrator nun die Vorteile der Tools für die elektronische Softwareverteilung nutzen.</span><span class="sxs-lookup"><span data-stu-id="d367d-117">To deploy a MED-V workspace, an administrator is now able to take advantage of their electronic software distribution tools.</span></span> <span data-ttu-id="d367d-118">Die in Med-v 1,0 verfügbare Client-Pull-Methode wurde entfernt, und der Med-v-Arbeitsbereich wird jetzt mithilfe von Methoden außerhalb von Med-v bereitgestellt.</span><span class="sxs-lookup"><span data-stu-id="d367d-118">The client-pull method available in MED-V 1.0 was removed and the MED-V workspace is now delivered by using methods outside MED-V.</span></span> <span data-ttu-id="d367d-119">Administratoren können Med-v-Arbeitsbereiche wie jedes andere Anwendungspaket behandeln und Bereitstellungen und Installationen von Med-v mithilfe der vorhandenen Tools und Prozesse planen.</span><span class="sxs-lookup"><span data-stu-id="d367d-119">Administrators can treat MED-V workspaces as they would any other application package and can schedule deployments and installations of MED-V by using their existing tools and processes.</span></span> <span data-ttu-id="d367d-120">MED-V-Installationen können lautlos bereitgestellt werden und können problemlos innerhalb einer vorhandenen Softwareverteilungsinfrastruktur verwaltet werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-120">MED-V installations can be deployed silently and can easily be managed inside an existing software distribution infrastructure.</span></span>

### <span data-ttu-id="d367d-121">MED-V-Arbeitsbereichsverwaltung</span><span class="sxs-lookup"><span data-stu-id="d367d-121">MED-V Workspace Management</span></span>

<span data-ttu-id="d367d-122">Der Med-v-Arbeitsbereich in Med-v 2,0 basiert auf einer virtuellen Windows Virtual PC-Festplatte.</span><span class="sxs-lookup"><span data-stu-id="d367d-122">The MED-V workspace in MED-V 2.0 is based on a Windows Virtual PC virtual hard disk.</span></span> <span data-ttu-id="d367d-123">Med-v hat die von Windows Virtual PC bereitgestellten Funktionen erweitert, indem die nahtlose Benutzeroberfläche verbessert wird, ohne dass Verschlüsselung oder spezielle Tools für den Zugriff auf den MED-V-Arbeitsbereich erforderlich sind.</span><span class="sxs-lookup"><span data-stu-id="d367d-123">MED-V has extended the capabilities that Windows Virtual PC provides by improving the seamless experience without requiring encryption or special tools to access the MED-V workspace.</span></span>

<span data-ttu-id="d367d-124">Nachdem Med-v auf einer Workstation bereitgestellt wurde, kann der Med-v-Arbeitsbereich mithilfe von Windows Virtual PC im Vollbildmodus geöffnet werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-124">After MED-V is deployed to a workstation, the MED-V workspace can be opened in full-screen mode by using Windows Virtual PC.</span></span> <span data-ttu-id="d367d-125">Mit dieser neuen Funktionalität wurde die Anforderung für Richtlinien entfernt, die eine Einstellung für Seamless-oder Vollbild-Modi festlegten, und auch die Notwendigkeit, den Vollbildmodus für Diagnose und Problembehandlung zu erzwingen, wurde entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-125">This new functionality removed the requirement for policies that set a preference for seamless or full-screen modes and also removed the need to force full-screen for diagnostics and troubleshooting.</span></span>

<span data-ttu-id="d367d-126">Das Veröffentlichen von Anwendungen im MED-V-Arbeitsbereich wird nicht mehr mit Profilen durchgeführt und manuell in den Pfad zu Anwendungen eingegeben.</span><span class="sxs-lookup"><span data-stu-id="d367d-126">Publishing applications to the MED-V workspace is no longer performed with profiles and by manually entering the path to applications.</span></span> <span data-ttu-id="d367d-127">Stattdessen tritt sie automatisch auf, wenn Anwendungen auf dem Gast installiert werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-127">Instead, it occurs automatically as applications are installed on the guest.</span></span> <span data-ttu-id="d367d-128">Das zentrale Bild-Repository, in dem Versionen der Bilder enthalten sind, die über die Trimm Übertragung übermittelt wurden, wird entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-128">The central image repository that included versions of the images that were delivered through trim transfer is removed.</span></span> <span data-ttu-id="d367d-129">Stattdessen ermöglicht Med-v Administratoren das Verwalten des Med-v-Arbeitsbereichs wie ein physischer Computer, indem Anwendungen und Updates ohne die Komplexität einer dedizierten MED-V-Infrastruktur verteilt werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-129">Instead, MED-V enables administrators to manage the MED-V workspace as they would a physical computer, by letting applications and updates be distributed without the complexity of a dedicated MED-V infrastructure.</span></span>

<span data-ttu-id="d367d-131">In einigen wichtigen Bereichen von MED-V 2,0 werden Verbesserungen oder Ergänzungen der folgenden Features widergespiegelt.</span><span class="sxs-lookup"><span data-stu-id="d367d-131">Several key areas of MED-V 2.0 reflect improvements or additions to the following features.</span></span>

### <span data-ttu-id="d367d-132">Erstellen eines MED-V-Arbeitsbereichs</span><span class="sxs-lookup"><span data-stu-id="d367d-132">MED-V Workspace Creation</span></span>

<span data-ttu-id="d367d-133">MED-V-Arbeitsbereiche müssen mithilfe von Windows Virtual PC erstellt werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-133">MED-V workspaces must be created by using Windows Virtual PC.</span></span> <span data-ttu-id="d367d-134">Vorhandene Virtual PC 2007-Bilder müssen migriert werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-134">Existing Virtual PC 2007 images must be migrated.</span></span> <span data-ttu-id="d367d-135">Das Tool für die Vorbereitung virtueller Computer ist nicht in Med-v 2,0 enthalten, und Administratoren sollten Ihre Bilder entsprechend der Med-v 2,0-Hilfedatei konfigurieren, aktualisieren und optimieren.</span><span class="sxs-lookup"><span data-stu-id="d367d-135">The virtual machine Prep tool is not included in MED-V 2.0 and administrators should configure, update, and optimize their images according to the MED-V 2.0 Help file.</span></span> <span data-ttu-id="d367d-136">Das Ausführen von Sysprep auf dem MED-V-Abbild ist ein erforderlicher Schritt und muss vor dem Verpacken ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-136">Running Sysprep on the MED-V image is a required step and must be performed before packaging.</span></span>

### <span data-ttu-id="d367d-137">MED-V-Arbeitsbereichs Verpackung</span><span class="sxs-lookup"><span data-stu-id="d367d-137">MED-V Workspace Packaging</span></span>

<span data-ttu-id="d367d-138">Windows PowerShell ist die Grundlage des MED-V-Arbeitsbereichs Pakets.</span><span class="sxs-lookup"><span data-stu-id="d367d-138">Windows PowerShell is the foundation of the MED-V Workspace Packager.</span></span> <span data-ttu-id="d367d-139">Diese Funktion ersetzt einige frühere Konsolen Fähigkeiten und-Funktionen, die zentralisierte Funktionen von MED-V verwaltet haben.</span><span class="sxs-lookup"><span data-stu-id="d367d-139">This functionality replaces some former console abilities and functionality that managed centralized functions of MED-V.</span></span> <span data-ttu-id="d367d-140">Der MED-V Workspace-Paketer verpackt lediglich die virtuelle Festplatte mit den entsprechenden Einstellungen und dem Bild, sodass Sie von Administratoren problemlos bereitgestellt werden kann.</span><span class="sxs-lookup"><span data-stu-id="d367d-140">The MED-V Workspace Packager merely packages the virtual hard disk with the appropriate settings and image so that it can be easily deployed by administrators.</span></span> <span data-ttu-id="d367d-141">Erweiterte Features werden mithilfe von Windows PowerShell bereitgestellt.</span><span class="sxs-lookup"><span data-stu-id="d367d-141">Advanced features are provided by using Windows PowerShell.</span></span>

### <span data-ttu-id="d367d-142">MED-V-Arbeitsbereichs Verteilung</span><span class="sxs-lookup"><span data-stu-id="d367d-142">MED-V Workspace Distribution</span></span>

<span data-ttu-id="d367d-143">Für med-v 2,0 ist keine dedizierte Serverinfrastruktur mehr erforderlich, und die Client Pull-Methode zum Bereitstellen von Med-v-Arbeitsbereichen wurde entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-143">Dedicated server infrastructure is no longer required for MED-V 2.0 and the client pull method to deploy MED-V workspaces was removed.</span></span> <span data-ttu-id="d367d-144">MED-V-Arbeitsbereiche werden jetzt mithilfe Ihrer Infrastruktur für die elektronische Softwareverteilung bereitgestellt und können auf allgemeinen Freigaben gespeichert werden, die für andere Installationspakete verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-144">MED-V workspaces are now deployed using your electronic software distribution infrastructure and can be stored on common shares that are used for other installation packages.</span></span>

### <span data-ttu-id="d367d-145">Erstmaliges Einrichten</span><span class="sxs-lookup"><span data-stu-id="d367d-145">First Time Setup</span></span>

<span data-ttu-id="d367d-146">Der erstmalige Setupprozess ist jetzt in die Standard-Imaging-Konvention von Sysprep integriert.</span><span class="sxs-lookup"><span data-stu-id="d367d-146">The first time setup process is now integrated with the standard imaging convention of Sysprep.</span></span> <span data-ttu-id="d367d-147">Beim erstmaligen Einrichten des Med-v-Arbeitsbereichs können die im Med-v-Arbeitsbereichs Paket angegebenen Einstellungen dynamisch auf das Bild angewendet werden, während das Mini Setup beginnt.</span><span class="sxs-lookup"><span data-stu-id="d367d-147">The MED-V workspace first time setup process can dynamically apply settings specified in the MED-V Workspace Packager to the image as it begins Mini-Setup.</span></span> <span data-ttu-id="d367d-148">Das Skript Tool in der Konsole wurde entfernt, und der erstmalige Setup-Vorgang basiert nun auf Optionen, die vom Administrator im MED-V Workspace-Paketer konfiguriert sind.</span><span class="sxs-lookup"><span data-stu-id="d367d-148">The scripting tool in the console was removed and the first time setup process is now based on options that are configured in the MED-V Workspace Packager by the administrator.</span></span>

### <span data-ttu-id="d367d-149">Anwendungsveröffentlichung</span><span class="sxs-lookup"><span data-stu-id="d367d-149">Application Publishing</span></span>

<span data-ttu-id="d367d-150">Administratoren können Anwendungen auf dem Med-v-Abbild entweder vor dem Verpacken, nach der Bereitstellung des Med-v-Arbeitsbereichs oder mithilfe einer Kombination aus beidem installieren.</span><span class="sxs-lookup"><span data-stu-id="d367d-150">Administrators can install applications on the MED-V image either before packaging, after the MED-V workspace is deployed, or by using a combination of both.</span></span> <span data-ttu-id="d367d-151">Med-v untersucht nicht mehr die Med-v Workspace-Richtlinie, um Anwendungen zu veröffentlichen, sondern bezieht sich stattdessen auf das, was tatsächlich auf dem Gast installiert ist.</span><span class="sxs-lookup"><span data-stu-id="d367d-151">MED-V no longer examines MED-V workspace policy to publish applications, but instead refers to what is actually installed on the guest.</span></span> <span data-ttu-id="d367d-152">Wenn Anwendungen auf dem Gast installiert sind, werden Sie automatisch erkannt und **im Startmenü des Hosts veröffentlicht** und können vom Endbenutzer gestartet werden.</span><span class="sxs-lookup"><span data-stu-id="d367d-152">As applications are installed on the guest, they are automatically detected and published to the host **Start** menu and are ready to be started by the end user.</span></span>

### <span data-ttu-id="d367d-153">URL-Umleitung</span><span class="sxs-lookup"><span data-stu-id="d367d-153">URL Redirection</span></span>

<span data-ttu-id="d367d-154">MED-V 2,0 bietet eine nahtlose Umleitung von Host-zu-Guest-Webadressen basierend auf den vom Administrator konfigurierten und verwalteten Richtlinien.</span><span class="sxs-lookup"><span data-stu-id="d367d-154">MED-V 2.0 provides seamless host-to-guest web address redirection based on the policies configured and managed by the administrator.</span></span> <span data-ttu-id="d367d-155">Nachdem eine URL an den Gast Browser umgeleitet wurde, besteht die standardmäßige Erfahrung darin, den Benutzer auf die umgeleitete Website zu beschränken.</span><span class="sxs-lookup"><span data-stu-id="d367d-155">After a URL is redirected to the guest browser, the default experience is to attempt to limit the user to that redirected site.</span></span> <span data-ttu-id="d367d-156">Auf diese Weise werden die Browseraktivitäten minimiert, die ein Benutzer ausführen kann, die nicht vom Administrator vorgesehen sind.</span><span class="sxs-lookup"><span data-stu-id="d367d-156">This minimizes the browsing activities that a user can perform that are not intended by the administrator.</span></span> <span data-ttu-id="d367d-157">Die Guest-to-Host-Browserumleitung wurde entfernt.</span><span class="sxs-lookup"><span data-stu-id="d367d-157">Guest-to-host browser redirection was removed.</span></span>

<span data-ttu-id="d367d-161">Darüber hinaus werden die Protokolle nicht mehr lokal verschlüsselt und zentral protokolliert.</span><span class="sxs-lookup"><span data-stu-id="d367d-161">In addition,
