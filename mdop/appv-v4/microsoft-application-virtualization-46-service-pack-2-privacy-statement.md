## Spezifische Features


Im weiteren Verlauf dieses Dokuments werden die folgenden speziellen Funktionen behandelt:

**Hinweis**  Dieser Abschnitt ist in zwei Abschnitte unterteilt: (1) Features in allen Versionen von App-v und (2) Features in App-v 4.6 SP1 und höher.

 

### Microsoft-Fehlerberichterstattung

**Funktionsweise dieses Features:**

Die Microsoft-Fehlerberichterstattung bietet einen Dienst, der es Ihnen ermöglicht, Probleme zu melden, die Sie möglicherweise mit App-V an Microsoft haben, und Informationen zu erhalten, die Ihnen helfen können, derartige Probleme zu vermeiden oder zu beheben.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Informationen zu den von Microsoft-Fehlerberichterstattung gesammelten, verarbeiteten oder übertragenen Informationen finden Sie in den Datenschutzbestimmungen für Microsoft-Fehlerberichterstattung unter <https://go.microsoft.com/fwlink/?linkid=50293> .

**Verwendung der Informationen:**

Wir verwenden die Fehler Berichterstattungsdaten, um Kunden Probleme zu lösen und unsere Software und Dienste zu verbessern.

**Wahlmöglichkeiten/Kontrolle:**

App-V ändert Ihre Einstellungen für die Microsoft-Fehlerberichterstattung nicht. Wenn Sie die Fehlerberichterstattung zuvor aktiviert haben, wird Microsoft die Informationen zu den aufgetretenen Fehlern gesendet. Wenn Microsoft zusätzliche Daten zur Analyse des Problems benötigt, werden Sie aufgefordert, die Daten zu überprüfen und auszuwählen, ob Sie Sie senden möchten.App-V respektiert immer die Microsoft-Fehler Berichterstattungseinstellungen.

**Wichtige Informationen:**

Unternehmenskunden können mithilfe von Gruppenrichtlinien konfigurieren, wie sich die Microsoft-Fehlerberichterstattung auf ihren Computern verhält. Zu den Konfigurationsoptionen gehört die Möglichkeit, die Microsoft-Fehlerberichterstattung zu deaktivieren. Wenn Sie ein Administrator sind und Gruppenrichtlinien für die Microsoft-Fehlerberichterstattung konfigurieren möchten, finden Sie die technischen Details unter <https://technet.microsoft.com/library/cc709644.aspx> .

### Microsoft Update

**Funktionsweise dieses Features:**

Microsoft Update ist ein Dienst, der Windows-Updates sowie Updates für andere Microsoft-Software, includingApp-V, bereitstellt.Details dazu, welche Informationen gesammelt werden, wie Sie verwendet werden und wie Sie Ihre Einstellungen ändern, finden Sie in den Datenschutzbestimmungen zu Update Services unter <https://go.microsoft.com/fwlink/?linkid=50142> .

**Wahlmöglichkeiten/Kontrolle:**

Wenn Microsoft Update nicht aktiviert ist, können Sie sich während des Setups anmelden, und nachfolgende Überprüfungen auf Updates folgen dem computerweiten Zeitplan. Sie können diese Option über das Microsoft Update-Control Panel-Element aktualisieren.

### Sammlung/Übertragung von Konfigurationsdaten

**Funktionsweise dieses Features:**

Das Produkt sammelt verschiedene Konfigurationselemente, einschließlich UserID, Machine-ID und SecurityGroup-Details, die in der Lage sind, Einstellungen für verwaltete Knoten durchzusetzen. Die Daten werden in der APP-v SQL-Datenbank gespeichert und über den App-v-Server und die Clientkomponenten übertragen, um die Konfiguration auf dem verwalteten Knoten durchzusetzen.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Benutzer-und Computer Informationen und Konfigurationsinhalte

**Verwendung der Informationen:**

Die Informationen werden verwendet, um die Anwendungszugriffs Konfiguration auf den verwalteten Knoten innerhalb des Unternehmens zu erzwingen. Die Informationen werden vom Unternehmen nicht verbleibt.

**Wahlmöglichkeiten/Kontrolle:**

Standardmäßig enthält das Produkt keine Daten. Alle Daten werden vom Administrator eingegeben und aktiviert und können in der Verwaltungskonsole angezeigt werden. Das Feature kann nicht deaktiviert werden, da es sich um die Produktfunktionalität handelt. Um dies zu deaktivieren, muss App-V deinstalliert werden.

**Wichtige Informationen:**

Keine dieser Informationen wird aus dem Unternehmen versandt.

### Informationen zum Paketverlauf

**Funktionsweise dieses Features:**

Sie erfasst den Paketverlauf und die Ressourceninformationen als Teil des Pakets.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Informationen zum Paket und zur Sequenz Umgebung werden gesammelt und während der Sequenzierung im Paketmanifest gespeichert.

**Verwendung der Informationen:**

Die Informationen werden vom Administrator verwendet, um die Aktualisierungen zu überwachen, die an einem Paket während des Lebenszyklus durchgeführt wurden. Sie wird auch von Software Bereitstellungssystemen verwendet, um die Paket Bereitstellungen innerhalb der Organisation zu überwachen.

**Wahlmöglichkeiten/Kontrolle:**

Dieses Feature ist immer aktiviert und kann nicht deaktiviert werden.

**Wichtige Informationen:**

Diese Administrator Informationen werden im Paket gespeichert und können von den Endbenutzern angezeigt werden, wenn Sie auf die XML-Datei zugreifen, die der Anwendung zugeordnet ist.

### Sammlung und Übertragung von Berichtsdaten

**Funktionsweise dieses Features:**

Das Produkt sammelt eine Vielzahl von Berichterstattungsdaten Punkten, einschließlich des Nutzernamens, um Berichte über die Verwendung des Produkts zu ermöglichen.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Informationen über die Verwendung von Computern, Paketen und Anwendungen werden auf jedem Computer erfasst, auf dem Reporting aktiviert ist.

**Verwendung der Informationen:**

Die Informationen werden verwendet, um die Verwendung der Anwendung innerhalb des Unternehmens zu melden. Die Informationen werden vom Unternehmen nicht verbleibt.

**Wahlmöglichkeiten/Kontrolle:**

Standardmäßig enthält das Produkt keine Daten. Daten werden nur erfasst, wenn das Berichterstattungsfeature auf dem App-V-Client aktiviert ist. Zum Deaktivieren der Sammlung von Berichtsdaten muss das Berichterstattungsfeature auf allen Clients deaktiviert sein.

**Wichtige Informationen:**

Keine dieser Informationen wird aus dem Unternehmen versandt.

### Features in App-V 4,6 SP1 und höher

In diesem Abschnitt werden bestimmte Features behandelt, die in App-V 4,6 SP1 und höher verfügbar sind.

### Programm zur Verbesserung der Benutzerfreundlichkeit

**Funktionsweise dieses Features:**

Das Programm zur Verbesserung der Benutzerfreundlichkeit ("CEIP") sammelt grundlegende Informationen zu Ihrer Hardwarekonfiguration und zur Verwendung unserer Software und Dienste, um Trends und Nutzungsmuster zu erkennen. CEIP sammelt auch den Typ und die Anzahl der auftretenden Fehler, die Software-und Hardwareleistung sowie die Geschwindigkeit der Dienste. Wir werden Ihren Namen, Ihre Adresse oder andere Kontaktinformationen nicht erfassen.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Weitere Informationen zu den von CEIP gesammelten, verarbeiteten oder übertragenen Informationen finden Sie in der CEIP-Datenschutzerklärung unter <https://go.microsoft.com/fwlink/?LinkID=52097> .

**Verwendung der Informationen:**

Wir verwenden diese Informationen, um die Qualität, Zuverlässigkeit und Leistung von Microsoft-Software und-Diensten zu verbessern.

**Wahlmöglichkeiten/Kontrolle:**

CEIP ist optional, und der Opt-in-Status kann während der Installation oder nach der Installation über die Benutzeroberfläche aktualisiert werden.  

### Anwendungspaket Beschleuniger

**Funktionsweise dieses Features:**

Kunden können Anwendungspaket Beschleuniger verwenden, um komplexe Anwendungen automatisch zu verpacken, ohne die Anwendung zu installieren. Mit dem App-V-Sequenzer können Sie Paket Beschleuniger für jedes virtuelle Paket erstellen. Sie können diese Paket Beschleuniger dann verwenden, um das gleiche virtuelle Paket in Zukunft automatisch neu zu erstellen. Sie können auch von Microsoft oder anderen Drittanbietern freigegebene Paket Beschleuniger verwenden, um das Verpacken komplexer Anwendungen zu vereinfachen und zu automatisieren.

**Gesammelte, verarbeitete oder übertragene Informationen:**

Anwendungspaket Beschleuniger können Informationen wie Computernamen, Benutzerkontoinformationen und Informationen zu Anwendungen enthalten, die in der Paket Beschleunigerdatei enthalten sind.

Wenn Sie beabsichtigen, Anwendungspaket Beschleuniger für Personen außerhalb Ihrer Organisation freizugeben, sollten Sie alle Einstellungen überprüfen und sicherstellen, dass die Paket Beschleuniger keine persönlichen oder Unternehmensinformationen enthalten. Sie können die Inhalte anzeigen, indem Sie die Paket Beschleunigerdateien mit einem beliebigen XML-Viewer öffnen. Im folgenden finden Sie Möglichkeiten, wie Sie Computer-oder Benutzerinformationen aus den Paket Beschleunigerdateien anzeigen und entfernen können, bevor Sie Sie für Personen außerhalb Ihres Unternehmens freigeben:

-   **Benutzername** : Wenn Sie sich bei dem Computer anmelden, auf dem der App-V-Sequencer ausgeführt wird, sollten Sie ein allgemeines Benutzerkonto wie **Administrator**verwenden. Sie sollten kein Konto verwenden, das auf einem vorhandenen Nutzernamen basiert.

-   **Computername**– geben Sie einen allgemeinen, nicht identifizierbaren Namen für den Computer an, auf dem der App-V-Sequencer ausgeführt wird.

-   **Server-URL**– verwenden Sie die Standardeinstellungen für die Server-URL-Konfigurationsinformationen auf der Registerkarte **Bereitstellung** in der App-V-Sequenzer-Konsole.

-   **Anwendungen** – Wenn Sie die Liste der Anwendungen, die auf dem Computer installiert wurden, auf dem der Sequencer ausgeführt wurde, beim Erstellen des Paket Beschleunigers nicht freigeben möchten, müssen Sie die **appv\_manifest.xml** -Datei löschen. Diese Datei befindet sich im Paketstammverzeichnis des virtuellen Anwendungspakets.

Es werden keine Informationen über die Verwendung der APP-Paket Beschleuniger Funktion durch die Kunden an Microsoft gesendet.
