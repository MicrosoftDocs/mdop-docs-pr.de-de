## <span data-ttu-id="35acc-123">Bekannte Probleme mit AGPM 4,0</span><span class="sxs-lookup"><span data-stu-id="35acc-123">Known issues with AGPM 4.0</span></span>


### <span data-ttu-id="35acc-124">Befehl ' aus Produktion importieren ' importiert keine Einstellungen in ein GPO, das ausgecheckt ist.</span><span class="sxs-lookup"><span data-stu-id="35acc-124">Import from Production command does not import settings into a GPO that is checked out</span></span>

<span data-ttu-id="35acc-125">Wenn Sie ein GPO in der Produktionsumgebung bearbeiten, müssen Sie das Gruppenrichtlinienobjekt aus der Produktion importieren, um das Gruppenrichtlinienobjekt im Offline Archiv zu aktualisieren.</span><span class="sxs-lookup"><span data-stu-id="35acc-125">If you edit a GPO in the production environment, you must import the GPO from production to update the GPO in the offline archive.</span></span> <span data-ttu-id="35acc-126">Mit dem Befehl " **aus Produktion importieren** " können Sie eine endgültige Produktionssicherung durchführen, bevor Sie die Bearbeitung abgeschlossen haben, sodass Sie bei Bedarf ein Rollback zur Produktionssicherung durchführen können.</span><span class="sxs-lookup"><span data-stu-id="35acc-126">The **Import from Production** command is intended to let you perform a final production backup before you finish editing so that you can roll back to the production backup if it is necessary.</span></span>

<span data-ttu-id="35acc-127">Wenn das Gruppenrichtlinienobjekt ausgecheckt ist, wenn Sie den Befehl **aus Produktion importieren** ausführen, werden die Produktionsänderungen nicht in die ausgecheckte Version des Gruppenrichtlinienobjekts übernommen.</span><span class="sxs-lookup"><span data-stu-id="35acc-127">If the GPO is checked out when you run the **Import from Production** command, then the production changes are not incorporated into the checked out version of the GPO.</span></span> <span data-ttu-id="35acc-128">Die importierte Version des Gruppenrichtlinienobjekts wird jedoch dem Verlauf des Gruppenrichtlinienobjekts hinzugefügt, obwohl diese Version nicht zur Bearbeitung zur Verfügung steht.</span><span class="sxs-lookup"><span data-stu-id="35acc-128">However, the imported version of the GPO is added to the history of the GPO even though that version is not available to be edited.</span></span> <span data-ttu-id="35acc-129">Wenn das Gruppenrichtlinienobjekt eingecheckt ist, ersetzt diese Version die importierte Version im Archiv, aber beide sind im Verlauf des Gruppenrichtlinienobjekts verfügbar.</span><span class="sxs-lookup"><span data-stu-id="35acc-129">When the GPO is checked in, that version will supersede the imported version in the archive, but both are available in the history of the GPO.</span></span>

<span data-ttu-id="35acc-130">**Problemumgehung:** Stellen Sie sicher, dass das Gruppenrichtlinienobjekt eingecheckt ist, bevor Sie es aus Production importieren.</span><span class="sxs-lookup"><span data-stu-id="35acc-130">**Workaround:** Make sure that the GPO is checked in before you import it from production.</span></span> <span data-ttu-id="35acc-131">Wenn das Gruppenrichtlinienobjekt nicht eingecheckt wurde, bevor Sie es importiert haben, können Sie den Befehl **Auschecken rückgängig** verwenden, um Ihre Änderungen zu verwerfen und die Version des Gruppenrichtlinienobjekts zurückzusetzen, das Sie aus der Produktionsumgebung importiert haben.</span><span class="sxs-lookup"><span data-stu-id="35acc-131">If the GPO was not checked in before you imported it, you can use the **Undo Check Out** command to discard your changes and roll back to the version of the GPO that you imported from production.</span></span>

### <span data-ttu-id="35acc-132">Ausgecheckte GPOs können in einer Umgebung mit einer Active Directory-Topologie mit mehreren Standorten nicht für mehrere Minuten bearbeitet werden</span><span class="sxs-lookup"><span data-stu-id="35acc-132">Checked out GPOs cannot be edited for several minutes in an environment that uses a multiple site Active Directory topology</span></span>

<span data-ttu-id="35acc-133">AGPM verwendet ein Client/Server-Modell.</span><span class="sxs-lookup"><span data-stu-id="35acc-133">AGPM uses a client/server model.</span></span> <span data-ttu-id="35acc-134">Der AGPM-Server und der AGPM-Client bestimmen jeweils seinen eigenen nächstgelegenen Domänencontroller für Gruppenrichtlinienvorgänge.</span><span class="sxs-lookup"><span data-stu-id="35acc-134">The AGPM Server and the AGPM Client each determine their own closest domain controller for Group Policy operations.</span></span> <span data-ttu-id="35acc-135">Wenn Sie ein GPO mithilfe eines AGPM-Clients Auschecken, überprüft der AGPM-Server das Gruppenrichtlinienobjekt aus dem Offline Archiv in einen temporären Ordner im Ordner "SYSVOL".</span><span class="sxs-lookup"><span data-stu-id="35acc-135">When you check out a GPO by using an AGPM Client, it is actually the AGPM Server that checks the GPO out from the offline archive to a temporary folder in the SYSVOL folder.</span></span>

<span data-ttu-id="35acc-136">Wenn sich der AGPM-Server und der AGPM-Client an verschiedenen Standorten befinden, ist das temporäre ausgecheckte Gruppenrichtlinienobjekt auf dem Domänencontroller der lokalen Website für mehrere Minuten oder bis zu 30 Minuten aufgrund der SYSVOL-Replikationswartezeit möglicherweise nicht vorhanden.</span><span class="sxs-lookup"><span data-stu-id="35acc-136">If the AGPM Server and the AGPM Client are in different sites, then the temporary checked out GPO may not be present on the local site's domain controller for several minutes or up to 30 minutes due to SYSVOL replication latency.</span></span> <span data-ttu-id="35acc-137">In diesem Fall können Sie das ausgecheckte Gruppenrichtlinienobjekt nicht mit der GPMC auf einem AGPM-Client bearbeiten, bis die SYSVOL-Replikation des ausgecheckten Gruppenrichtlinienobjekts stattgefunden hat.</span><span class="sxs-lookup"><span data-stu-id="35acc-137">In this situation, you cannot edit the checked out GPO using the GPMC on an AGPM Client until SYSVOL replication of the checked out GPO has occurred.</span></span>

<span data-ttu-id="35acc-138">**Problemumgehung:** Als bewährte Methode sollten Sie AGPM-Clients am gleichen Standort wie der AGPM-Server positionieren, mit dem Sie eine Verbindung herstellen, damit Sie nicht warten müssen, bis die SYSVOL-Replikation erfolgt, bevor Sie ein ausgechecktes Gruppenrichtlinienobjekt bearbeiten können.</span><span class="sxs-lookup"><span data-stu-id="35acc-138">**Workaround:** As a best practice, you should position AGPM Clients in the same site as the AGPM Server to which they connect so that you do not have to wait for SYSVOL replication to occur before you can edit a checked out GPO.</span></span>

### <span data-ttu-id="35acc-139">AGPM kann den Sicherungs Grenzwert nicht lesen, wenn Ihr Konto nicht über die Berechtigungen für das Archiv verfügt</span><span class="sxs-lookup"><span data-stu-id="35acc-139">AGPM cannot read the backup limit if your account does not have permissions for the archive</span></span>

<span data-ttu-id="35acc-140">Wenn Sie sich bei einem AGPM-Client mit einem Konto anmelden, dem keine Berechtigungen für das AGPM-Archiv zugewiesen wurden, starten Sie die Gruppenrichtlinien-Verwaltungskonsole (Group Policy Management Console, GPMC), und klicken Sie dann auf **Steuerelement ändern**, wenn Sie die folgende Fehlermeldung erhalten.</span><span class="sxs-lookup"><span data-stu-id="35acc-140">On an AGPM Client, if you log on by using an account that has not been delegated permissions to the AGPM archive, start the Group Policy Management Console (GPMC), and then click **Change Control**, you receive the following error.</span></span>

``` syntax
Failed to read backup purge limit for this domain. 

The following error occurred: 
You do not have sufficient permissions to perform this operation. 
Microsoft.Agpm.AccessDeniedException (80070005)
```

<span data-ttu-id="35acc-141">**Problemumgehung:** Wenden Sie sich an einen AGPM-Administrator (Vollzugriff), und fordern Sie die Stellvertretung des Zugriffs auf AGPM für Ihr Konto an.</span><span class="sxs-lookup"><span data-stu-id="35acc-141">**Workaround:** Contact an AGPM Administrator (Full Control) and request that they delegate access to AGPM for your account.</span></span> <span data-ttu-id="35acc-142">Wenn Sie ein AGPM-Administrator sind, melden Sie sich mit einem Konto an, dem die AGPM-Administratorrolle zugewiesen ist, damit Sie den Zugriff für das zusätzliche Konto delegieren können.</span><span class="sxs-lookup"><span data-stu-id="35acc-142">If you are an AGPM Administrator, log on by using an account to which the AGPM Administrator role is assigned so that you can delegate access for the additional account.</span></span> <span data-ttu-id="35acc-143">Weitere Informationen finden Sie in der Hilfe zu AGPM unter "Delegieren des Zugriffs auf das Archiv auf Domänenebene".</span><span class="sxs-lookup"><span data-stu-id="35acc-143">For more information, see "Delegate Domain-Level Access to the Archive" in the AGPM Help.</span></span>
