# Application Virtualization-Glossar


<table>
<colgroup>
<col width="50%" />
<col width="50%" />
</colgroup>
<thead>
<tr class="header">
<th align="left">Begriff</th>
<th align="left">Definition</th>
</tr>
</thead>
<tbody>
<tr class="odd">
<td align="left"><p>Aktives Upgrade</p></td>
<td align="left"><p>Ein Upgrade, mit dem eine neue Version einer Anwendung zu einem App-V-Verwaltungsserver oder Streamingserver hinzugefügt werden kann, ohne dass sich dies auf Benutzer auswirkt, die die Anwendung gerade ausführen.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Stamm der Anwendungsquelle</p></td>
<td align="left"><p>Verweis auf einen Registrierungsschlüssel, der eine Überschreibung der OSD-CodeBase für das href-Element ermöglicht (beispielsweise den Quellspeicherort). Dieser Registrierungswert ermöglicht es einem Administrator oder einem ESD-System, sicherzustellen, dass das Laden der Anwendung nach einem geplanten Topologieverwaltung-Schema durchgeführt wird.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Application Virtualization-Laufwerk</p></td>
<td align="left"><p>Das standardmäßige Clientlaufwerk für virtuelle Anwendungen (Q:) aus welchen sequenzierten Anwendungen ausgeführt werden.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Application Virtualization-Berichterstellung</p></td>
<td align="left"><p>Für die Datenanalyse gesammelte Informationen zur virtuellen Anwendung. Daten werden für die Assemblierung von benutzerdefinierten Ansichten und Interpretationen erfasst.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Konfigurationsparameter für die Konfiguration</p></td>
<td align="left"><p>Ein Client-Laufzeitrichtlinien-Konfigurationsparameter, der es ermöglicht, den sekundären Feature-Block einer virtualisierten Anwendung automatisch im Hintergrund an den Client zu streamen.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Verzweigen eines Pakets</p></td>
<td align="left"><p>Zum Aktualisieren eines vorhandenen sequenzierten Anwendungspakets und zum parallelen Ausführen des ursprünglichen sequenzierten Anwendungspakets.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Dynamische Suite-Komposition</p></td>
<td align="left"><p>Eine Application Virtualization-Funktion, die es einem virtuellen Anwendungspaket ermöglicht, abhängige Plug-ins oder Middleware-Pakete zur Verwendung der virtuellen Umgebung zu ermöglichen. Dieses Feature ermöglicht es Plug-ins und Middleware-Paketen, die Registrierungseinstellungen des primären Pakets zu verwenden, und die Pakete Verhalten sich und interagieren miteinander auf die gleiche Weise, als wären Sie lokal auf einem entstummer installiert.</p></td>
</tr>
<tr class="even">
<td align="left"><p>ICO-Datei</p></td>
<td align="left"><p>Die Datei für das Symbol auf dem Desktop des Clients, mit dem eine sequenzierte Anwendung gestartet wird.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Symbol Quellstamm</p></td>
<td align="left"><p>Verweis auf einen Registrierungsschlüssel, der die Angabe eines Quellspeicherorts für den Symbol Abruf eines sequenzierten Anwendungspakets während der Veröffentlichung ermöglicht. Symbol Quell Stämme unterstützen nur UNC-Formate (keine URLs).</p></td>
</tr>
<tr class="even">
<td align="left"><p>Installationsverzeichnis</p></td>
<td align="left"><p>Das Verzeichnis, in dem das Installationsprogramm für Application Virtualization Sequencer seine Dateien platziert.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization-Desktop Client</p></td>
<td align="left"><p>Eine Anwendung, die sich auf einem Windows-basierten Computer Desktop befindet und die mit dem virtuellen Microsoft System Center-Anwendungs Server kommuniziert und authentifiziert, um den Anwendungscode zu empfangen und eine sequenzierte Anwendung lokal ausführen zu lassen.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization für Terminal Dienste</p></td>
<td align="left"><p>Das Application Virtualization-Feature, einschließlich Client-und Server Komponenten, die in einer Terminal Dienste-Umgebung ausgeführt werden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization-Verwaltungskonsole</p></td>
<td align="left"><p>Zentrales Snap-in für die Microsoft Management Console, mit dem Sie das Application Virtualization Management System verwalten.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization-Verwaltungs System</p></td>
<td align="left"><p>Eine bestimmte Bereitstellung der Application Virtualization-Plattform, die alle Komponenten umfasst, die von einem einzelnen Datenspeicher verwaltet werden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization Management Web Service</p></td>
<td align="left"><p>Ein Dienst, der als Vermittler zwischen der Application Virtualization-Verwaltungskonsole und dem Datenspeicher fungiert. Zuerst werden die Benutzer authentifiziert, und dann können autorisierte Administratoren Daten im Datenspeicher manipulieren.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization-Plattform</p></td>
<td align="left"><p>Name für die Microsoft-Produkte, die zum Erstellen, speichern, verteilen und Ausführen virtualisierter Anwendungen verwendet werden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft Application Virtualization-Sequenzer</p></td>
<td align="left"><p>Anwendung, die den Installations-und Einrichtungsprozess für Anwendungen überwacht und aufzeichnet, damit eine Anwendung sequenziert und in der virtuellen Umgebung ausgeführt werden kann.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft Application Virtualization-Terminaldiensteclient</p></td>
<td align="left"><p>Eine Anwendung, die sich auf einem Terminal Server befindet und die mit dem Microsoft Virtual Application Server kommuniziert und authentifiziert, um den Anwendungscode zu empfangen und eine sequenzierte Anwendung lokal auszuführen.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Microsoft System Center Application Virtualization-Verwaltungs Server</p></td>
<td align="left"><p>Einer von zwei Application Virtualization Server-Typen, aus denen ein sequenziertes Anwendungspaket gestreamt werden kann. Zusätzlich zu den virtuellen Streaming-Anwendungspaketen bietet der Application Virtualization Management Server weitere Dienste wie Veröffentlichung, Verwaltung, Berichterstellung usw.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Microsoft System Center Application Virtualization Streaming Server</p></td>
<td align="left"><p>Einer von zwei Application Virtualization Server-Typen, aus denen ein sequenziertes Anwendungspaket gestreamt werden kann. Der Streamingserver strömt Anwendungen nur auf die Clientcomputer über und bietet keine anderen Dienste wie Veröffentlichung, Verwaltung, Berichterstellung usw.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Open Software Descriptor (OSD)-Datei</p></td>
<td align="left"><p>Eine XML-basierte Datei, in der der Client angewiesen wird, wie die sequenzierte Anwendung vom Application Virtualization Management/Streaming Server abgerufen wird und wie die sequenzierte Anwendung in Ihrer virtuellen Umgebung ausgeführt wird.</p></td>
</tr>
<tr class="even">
<td align="left"><p>OSD-Quellstamm</p></td>
<td align="left"><p>Verweis auf einen Registrierungsschlüssel, der die Angabe eines Quellspeicherorts für das Abrufen von OSD-Dateien für ein Anwendungspaket während der Veröffentlichung ermöglicht. OSD-Quell Stämme unterstützen nur UNC-Formate (keine URLs).</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Paketstammverzeichnis</p></td>
<td align="left"><p>Das Verzeichnis auf dem Sequenzcomputer, auf dem die Dateien für das sequenzierte Anwendungspaket installiert sind. Dieses Verzeichnis ist auch virtuell auf dem Computer vorhanden, auf den eine sequenzierte Anwendung gestreamt wird.</p></td>
</tr>
<tr class="even">
<td align="left"><p>primärer Feature-Block</p></td>
<td align="left"><p>Der minimale Inhalt in einem Anwendungspaket, der für die Ausführung einer Anwendung erforderlich ist. Der Inhalt des primären Funktionsblocks wird während der Anwendungsphase der Sequenzierung identifiziert und besteht in der Regel aus dem Inhalt der am häufigsten verwendeten Anwendungsfeatures.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>sekundärer Feature-Block</p></td>
<td align="left"><p>Der Rest des Anwendungspakets, das nicht im primären Feature-Block enthalten ist. Dieser Inhalt wird bei Bedarf an den Client gestreamt, wenn Anwendungsfeatures verwendet werden.</p></td>
</tr>
<tr class="even">
<td align="left"><p>sequenzierte Anwendung</p></td>
<td align="left"><p>Eine Anwendung, die vom Sequencer überwacht wurde, aufgeteilt in primäre und sekundäre Funktionsblöcke, gestreamt an einen Computer, auf dem der Microsoft Application Virtualization Terminal Services-Client oder der Microsoft Application Virtualization-Desktop Client ausgeführt wird, und kann innerhalb der eigenen virtuellen Umgebung ausgeführt werden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>sequenziertes Anwendungspaket</p></td>
<td align="left"><p>Die Dateien, die eine virtuelle Anwendung enthalten und die Ausführung einer virtuellen Anwendung ermöglichen. Diese Dateien werden nach der Sequenzierung erstellt und umfassen insbesondere OSD-, SFT-, SPRJ-und ICO-Dateien.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Sequenzierung</p></td>
<td align="left"><p>Der Vorgang zum Erstellen eines Anwendungspakets mithilfe von Application Virtualization Sequencer. In diesem Prozess wird eine Anwendung überwacht, Ihre Tastenkombinationen konfiguriert und ein sequenziertes Anwendungspaket mit den OSD-, SFT-, SPRJ-und ICO-Dateien erstellt.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Sequenzcomputer</p></td>
<td align="left"><p>Der Computer, der zum Durchführen der Sequenzierung und zum Erstellen eines sequenzierten Anwendungspakets verwendet wird.</p></td>
</tr>
<tr class="even">
<td align="left"><p>Sequenz-Assistent</p></td>
<td align="left"><p>Schritt-für-Schritt-Assistent, der den Benutzer durch die Sequenzierung einer Anwendung führt, einschließlich der Paketkonfiguration, der Installation der Anwendung oder der zu sequenzierten Anwendungen und der Sequenzierung des Anwendungspakets für Streaming.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>SFT</p></td>
<td align="left"><p>Eine Datei, die eine oder mehrere sequenzierte Anwendungen enthält, die der Sequencer in Streaming-Blöcke gepackt hat, sowie die zugehörigen Übermittlungsinformationen. Die SFT-Datei wird auf jedem Server gespeichert, der die verpackten Anwendungen an einen Client streamen muss.</p></td>
</tr>
<tr class="even">
<td align="left"><p>SFTMIME</p></td>
<td align="left"><p>Eine Befehlszeilenschnittstelle zum Einrichten und Verwalten von Anwendungen, Dateitypzuordnungen und Desktopkonfigurationsservern, die von Application Virtualization Desktop-und Terminal Dienste-Clients verwaltet werden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>SPRJ-Datei</p></td>
<td align="left"><p>Eine XML-basierte Sequenzer-Projektdatei, in der der Sequencer seine Ausschlusselemente speichert und die Elementinformationen analysiert. Die SPRJ-Datei wird bei der Erstellung von Anwendungsdatensätzen sowie beim Upgrade auf ein Paket stark verwendet.</p></td>
</tr>
<tr class="even">
<td align="left"><p>virtuelle Anwendung</p></td>
<td align="left"><p>Eine vom Sequencer verpackte Anwendung, die in einer eigenständigen virtuellen Umgebung ausgeführt werden soll. Die virtuelle Umgebung enthält die Informationen, die erforderlich sind, um die Anwendung auf dem Client auszuführen, ohne die Anwendung lokal zu installieren.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Virtuelles com</p></td>
<td align="left"><p>Das Subsystem, das COM-Objekte verwaltet, die von Anwendungsprozessen erstellt wurden, die in einer virtuellen Umgebung ausgeführt werden, und Konflikte mit denselben Objekten verhindert, die außerhalb der virtuellen Umgebung erstellt wurden.</p></td>
</tr>
<tr class="even">
<td align="left"><p>virtuelles Verzeichnis</p></td>
<td align="left"><p>Ein Opaque-Verzeichnis, in dem nur Dateien und Unterverzeichnisse, die im Anwendungspaket definiert oder durch Interaktion mit einer Anwendung in einer virtuellen Umgebung erstellt wurden, sichtbar sind. Alle Dateien in einem gleichnamigen lokalen Verzeichnis sind für die Anwendung nicht sichtbar.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>virtuelle Umgebung</p></td>
<td align="left"><p>Ein Laufzeitcontainer, der die verfügbaren Ressourcen für Anwendungsprozesse definiert, die aus einem sequenzierten Anwendungspaket gestartet wurden.</p></td>
</tr>
<tr class="even">
<td align="left"><p>virtuelle Datei</p></td>
<td align="left"><p>Einen Dateinamen in der virtuellen Umgebung, der einem alternativen Zielspeicherort zugeordnet ist. Eine virtuelle Datei wird neben anderen Dateien im enthaltenden Verzeichnis angezeigt, unabhängig davon, ob es sich um ein virtuelles oder lokales Verzeichnis handelt.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>virtuelles Dateisystem</p></td>
<td align="left"><p>Das Subsystem, das Dateisystemanforderungen von Anwendungsprozessen abfängt und umleitet, die in einer virtuellen Umgebung ausgeführt werden. Diese Anforderungen werden basierend auf den im Anwendungspaket definierten virtuellen Dateien und Verzeichnissen verarbeitet und durch die Interaktion mit einer virtuellen Anwendung erstellt oder geändert.</p></td>
</tr>
<tr class="even">
<td align="left"><p>virtuelle Registrierung</p></td>
<td align="left"><p>Das Subsystem, das Registrierungsanforderungen für Schlüssel und Werte aus Anwendungsprozessen abfängt und umleitet, die in einer virtuellen Umgebung ausgeführt werden. Die Umleitung basiert auf den im Anwendungspaket definierten Registrierungsinformationen, die durch die Interaktion mit einer virtuellen Anwendung erstellt oder geändert wurden.</p></td>
</tr>
<tr class="odd">
<td align="left"><p>Virtuelle Dienste</p></td>
<td align="left"><p>Das Subsystem, das als Dienststeuerungs-Manager (SCM) für Dienste fungiert, die in einer virtuellen Umgebung ausgeführt werden.</p></td>
</tr>
</tbody>
</table>
